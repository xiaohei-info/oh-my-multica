## Title

Welcome to oh-my-multica — ask questions, share deliveries, and help shape v1

## Body

oh-my-multica is a software delivery control layer built on Multica. Multica
provides the shared workspace, work items, runtimes, and persistent execution
history. oh-my-multica adds the path from a requirement to reviewed design,
acceptance criteria, a dynamically planned DAG, dependency-aware execution,
structured evidence, independent review, merge, and final acceptance.

The project is aimed at people who already use Coding Agents heavily and want
to spend less time supervising individual sessions. It is also useful when a
large amount of bounded implementation and testing work should be delegated to
cost-efficient models without handing project-completion judgment to a single
supervising Agent.

Please use:

- **Q&A** for installation, configuration, recovery, and architecture questions.
- **Ideas** for problems that do not yet have a clear implementation boundary.
- **Show and tell** for a real delivery, including what failed and how it recovered.
- **Issues** for reproducible product defects and focused feature requests.

The most useful feedback right now is concrete: where installation stopped,
which concept was hard to understand, what exit 20 asked you to decide, or what
kind of repository and Agent Team you tried. Sanitized failure reports are more
helpful than a generic “it works” or “it doesn’t work.”
