# oh-my-multica v1.0.0

The first public release of oh-my-multica adds a production-grade software
delivery control layer on top of Multica.

It combines Agent-authored design and dynamic DAG planning with a deterministic
Loop that owns dependency scheduling, evidence gates, independent review,
bounded rework, merge, recovery, and final acceptance.

## What is included

- Design → acceptance → manifest planning pipeline with independent review.
- Dependency-aware parallel delivery through Multica workspaces and runtimes.
- Structured verification evidence and reproducible Reviewer verdicts.
- Optional CI and Pull Request integration with bounded failure recovery.
- Final flow-based acceptance before the complete DAG can converge.
- Persistent execution state, explicit exit codes, and recovery commands.
- Human, Controller Agent, and local read-only web entry points.
- English and Simplified Chinese documentation and packaged Guides.
- Built-in Agent Team templates for common software engineering roles.

## Requirements

- Python 3.10 or later.
- A Git repository with at least one commit and a pushable `origin` remote.
- Multica CLI and connected Coding Agent runtimes when using the Multica engine.

See the repository README for installation and the shortest path to your first
delivery plan.

For concrete evidence, read the case study
[Building the v1 foundation on Multica](../case-studies/building-v1-on-multica.md)
and run the [local failure-and-recovery demo](../demo/README.md).

## Upgrade notes

This is the first public release. Existing internal projects using earlier
snapshots should run `omac init --check` and review their `.omac/config.yaml`
before continuing an existing manifest.
